A Regex Iterator

Contributed by Daniel C. Sobral
Source A Question on Scala-User
Tested with Scala version 2.9.2

What is the result of executing the following code?

val t = "this is a test"
val rx = " ".r
val m = rx.findAllIn(t)
println(m)
println(m.end)
println(rx.findAllIn(t).end)
  
  1. non-empty iterator
5
5

  2. empty iterator
java.lang.IllegalStateException: No match available

  3. non-empty iterator
5
java.lang.IllegalStateException: No match available

  4. non-empty iterator
java.lang.IllegalStateException: No match available

Explanation

The result of findAll is an Iterator. Until you somehow ask for or check the first element, such as calling next, hasNext or causing them to be called through toString, no regex match will be tried, causing the "end" method -- from MatchData, also extends by MatchIterator -- to throw an exception.
